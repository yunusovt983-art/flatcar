### 2. DDD-рефакторинг: Арки A–T (258–319) + W (342–344) — 65 эпиков

**Все 65 ✅ завершены.** Каждый заперт fitness-тестом (ratchet-храповик или семантический гейт).
Итог: гексагональная архитектура с 14 крейтами, принудительными DDD-границами, единым
`phase_to_command`+`build.sh`, единым `enum BuildPhase` в shared-kernel.

| Арка | Камни | Тема |
|------|-------|------|
| A | Core domain extraction | Shared Kernel + bc-build + bc-baremetal |
| B–C | Factory + Release | Декларативная фабрика, BOM/lock, SBOM |
| D | Unified PhaseResult | Один тип результата фазы |
| E | Event Sourcing | BuildRun как event-sourced агрегат |
| F | Omaha multi-arch | arm64 получает апдейты |
| G | phase_to_command | Единый диспетчер фаз через build.sh |
| H–J | Deploy/Mesh/OTA ports | DeployOrchestrator, HelmExecutor, OTA pipeline |
| K | Timeout-guard | Per-phase timeout в ScriptRunner |
| L | OTA delta + Ed25519 | delta_generator + подпись payload |
| M | Deploy refactor | Агрегат DeployOrchestrator |
| N–S | Вспомогательные | CQRS read-model, Specification pattern, BuildPlanFactory, ManifestPersistence, Event Sourcing replay |
| T | Ubiquitous Language | Глоссарий DDD-терминов в коде + UI |
| **W** | Анти-дрейф фаз | 3 skew-гейта (Rust↔TS↔CI) |
