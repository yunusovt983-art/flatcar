What's Verified Here vs Gated

Verified on arm64 Mac (cargo test + openssl + bash -n)

  • All DDD fitness gates (76 tests enforcing architectural invariants)
  • Pure Rust domain models (trust-chain, rollout FSM, supply-chain monitor, reproducibility)
  • CLI commands (verify-trust-chain, sc-monitor, repro-check, slsa)
  • Openssl sign/verify roundtrips (artifact signatures, verity roothash)
  • Bash syntax validation (all scripts pass bash -n)

Gated on amd64 Linux + QEMU/K8s/TPM

  • dm-verity boot, A/B rollback (gptprio), Secure Boot (OVMF)
  • Tetragon in-kernel enforce, real-time export
  • TPM measured boot, seal/unseal, remote attestation
  • Fleet update with live update_engine
  • Kubernetes mTLS/policy enforcement (Cilium + Istio ambient)
  • YDB/IPFS federation failover

Architecture

DDD hexagonal architecture with 14 Rust crates (bounded contexts), enforced by 76 fitness tests including ratchet-based burn-down of legacy I/O leaks. Domain logic is pure (no I/O); all external interactions through port traits with adapters.
